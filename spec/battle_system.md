# 救急・救命・救助知識学習RPG 仕様書：バトルシステム

## 1. バトル（救護アクション）の基本フロー

各ステージのバトルは、傷病者の発生から医療ギルド（現実の救急車・病院）への引き継ぎまでをシミュレートします。

1. **初期状況確認（アセスメントフェーズ）**
   * 傷病者のグラフィックと周囲の状況（倒れている場所、周囲の危険物、野次馬の有無など）が表示される。
2. **コマンド選択と実行（救護フェーズ）**
   * プレイヤーは表示されるコマンドから適切な行動を選択する。じっくり考えられるように、リアルタイムのタイマーではなく、**コマンド選択による時間消費制（思考時間重視型）**を採用する。
3. **結果反映と状況変化**
   * 正しい行動の場合：傷病者の容態が1段階回復する。
   * 誤った行動の場合：傷病者の容態が1段階悪化する（または「失敗回避」が発動し、未遂で留まる）。
4. **引き継ぎとクリア判定**
   * 傷病者の容態（ステータス）が最大値（5）に到達するか、適切な一連の処置が完了し、医療ギルドの馬車や救助隊が到着するとステージクリア。

## 2. 画面構成（スマホ・縦画面最適化）

スマートフォン画面（縦画面）における直感的なレイアウト構成：

* **上部: ステータス・情報領域**
  * **残り行動回数 / 猶予時間（タイムリミットバー）**: あと何回のアクション（コマンド選択）が許されているかをゲージで可視化。
  * **視覚的バイタルUI**: 
    * 傷病者の容態（5段階ステータス：1が危険、5が安定。初期値は3）をビジュアル（例：5つのセグメントに分割されたインジケーター、心拍グラフの簡易波形、酸素/呼吸ゲージ等）で表現。
* **中央: ビジュアル領域**
  * 傷病者と周囲の状況を示す2Dイラストやアニメーション（例：心肺停止時はバイタルUIが赤く点滅し、傷病者の胸の動きが止まっている様子を描く）。
  * 実行した処置の演出（エフェクト）や、バトルテキストログ（メッセージ）の表示。
* **下部: 操作・コマンド領域**
  * **選択コマンドボタン**: タップしやすいように、親指が届く範囲に大きくボタンを配置。
  * **ショートカットスロット**: 所持している魔道具（救急アイテム）のアイコン。長押しすることで、現実の対応知識（例：「AEDの役割」など）をいつでも確認できる。

## 3. コマンドシステムの詳細

### 3.1 共通コマンド
どのステージでも基本的に使用可能な基本コマンド：
1. **「周囲の安全確認」**: 現場の危険（崩落、二次災害など）を確認する。
2. **「意識の確認」**: 傷病者に声をかけ、意識があるか調べる。
3. **「応援要請」**: 周囲の人に大声で助けを求める。
4. **「呼吸の確認」**: 胸やお腹の動きを見て呼吸をしているか調べる。

### 3.2 特殊コマンド（魔道具・アイテムの使用）
所持しているアイテム（エルフの癒やし布、古代の蘇生魔石など）を使用するコマンド。アイテムが解放・所持されている場合のみ選択可能。

## 4. 特殊アクション：応援要請（NPC協力）

「応援要請」を選択した際、プレイヤーの進行状況に応じて協力の内容が変化します。

* **基本仕様**: 
  * 周囲の一般市民が、心肺蘇生のサポートや医療ギルドへの連絡を代行してくれ、タイムリミット（猶予時間）が延長される。
* **特殊NPCの登場（ゲームをある程度進めた場合）**:
  * ステージをある程度進めると、コマンド実行時に**低確率（15%以下。成長によって確率上昇するが上限は15%）**で特殊なNPCが登場する。
  * **兵士（重装甲）**: 
    * 【効果】: 不適切行動時のステータス（バイタル）減少（1段階減少）を1回だけ防ぐバリア（防御効果）を付与する。
  * **魔法使い**: 
    * 【効果】: 救助対象（傷病者）のステータス（バイタル）を1段階即座に回復（上昇）させる。

## 5. 判定とペナルティ・メッセージ

### 5.1 正誤判定とバイタル（ステータス）変化
* **バイタルの5段階仕様**:
  * 傷病者の容体（バイタルステータス）は **1 〜 5** の5段階で表される。
  * ステージ開始時の初期ステータスは **3**。
  * **5**に達すると、完全に容体が安定し「クリア」となる。
  * **1**は極めて危険な状態で、1のときにさらに不適切な行動をして減少すると「ゲームオーバー（救護失敗）」となる。
* **判定処理**:
  * **正しい行動（適切なコマンド）**: 100%成功。バイタルステータスが **1段階上昇**。
  * **不適切な行動（誤ったコマンド）**: バイタルステータスが **1段階減少**。

### 5.2 メッセージ・演出テキスト
コマンド選択時および容体変化時には、以下のメッセージがログエリアに明示的に表示されます。
* **正しいコマンド選択時**: 「正しい行動を選んだ」
* **不適切なコマンド選択時**: 「不適切な行動を選んだ」
* **バイタル上昇時（1段階上昇）**: 「容体が良くなった」
* **バイタル減少時（1段階減少）**: 「容体が悪化した」

### 5.3 失敗回避システム（成長要素）
プレイヤーの能力「冷静さ（マインドセット）」のステータス値に応じて、不適切なコマンドを選択した際に確率で「失敗回避」が発動。
* **発動時**: キャラクターの「待て、この状況でこの対処はかえって危険だ…！」といった内省メッセージとともに、アクションを実行せずにコマンド選択画面に戻る。この場合、タイムリミットやバイタルは減少せず、ペナルティも発生しない。

## 6. ゲームオーバー（敗北）条件

以下のいずれかの条件を満たすと敗北（ゲームオーバー）となります。

1. **タイムリミットゲージの枯渇**
   * コマンド実行によるタイムリミットの消費やペナルティが累積し、ゲージが完全にゼロになった場合。
2. **バイタルステータスのゼロ化**
   * バイタルステータスが1の状態で、不適切な処置をさらに行い、1段階下回った場合。
